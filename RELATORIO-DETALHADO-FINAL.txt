ENDPOINT 2: POST /api/gemini/gerar-tarefas
──────────────────────────────────────────
Verbo HTTP: POST
Autenticação: Header API_TOKEN + Body token_verificacao
Função: Recebe tarefas geradas pelo Gemini e salva no banco

Request esperado (JSON):
{
  "token_verificacao": "xyz123",
  "tarefas": [
    { titulo: "Ler capítulo 5", materia: "Português", dificuldade: "media" },
    { titulo: "Exercicios 1-10", materia: "Matemática", dificuldade: "alta" }
  ]
}

Processos:
1. Valida token
2. Para cada tarefa:
   - Insert em: tarefas (titulo, materia, dificuldade, criado_em, concluida=0)
3. Retorna: { sucesso: true, quantidade: 2, tarefas_criadas: [...] }

Responsabilidade:
- Receber dados de AI externa
- Validação básica
- Salvar no banco
- Retornar confirmação


ENDPOINT 3: POST /api/gemini/webhook
─────────────────────────────────────
Verbo HTTP: POST
Autenticação: Opcional (pode ser chamado por cron externo)

Uso: Pode ser chamado por:

- Outro serviço

Request:
{

}

Respostas:
- Se acao="gerar-tarefas" → Chama otherapi e salva
- Se acao="analisar-progresso" → Retorna dados do progresso
- Se outro valor → Erro 400

Evidência de Problema:

Usuário tentou deletar banco:
cd c:\Users\lucia\sac-1c & del data\database.sqlite* & echo OK

Resultado:
─────────
c:\Users\lucia\sac-1c\data\database.sqlite
O arquivo já está sendo usado por outro processo.

c:\Users\lucia\sac-1c\data\database.sqlite-shm  
O arquivo já está sendo usado por outro processo.

c:\Users\lucia\sac-1c\data\database.sqlite-wal
O arquivo já está sendo usado por outro processo.

OK


Diagnóstico:
────────────
❌ Arquivo não foi deletado
✓ Sistema retornou código de sucesso (OK print)
✗ Banco ainda existe e está LOCKED

Causa Raiz:
───────────
Processo Node.js running: node start.js
- Servindo na porta 3000
- Conexão ativa com banco.sqlite
- Lock exclusivo no arquivo

Windows não permite deletar arquivos em uso por outro aplicativo


Solução:
────────
METODO 1: Task Manager (Visual)
- Ctrl + Shift + Esc
- Procurar "node"
- End Task
- Depois deletar

METODO 2: PowerShell Command
- Stop-Process -Name node -Force
- Depois deletar

METODO 3: CMD Command
- taskkill /IM node.exe /F
- Depois deletar

METODO 4: File Explorer
- Fechar manualment terminal
- Explorador → data/ → deletar manualmente

Após matar processo:
- Arquivo = liberado
- Pode deletar
- Novo banco será criado na próxima execução de node start.js

GLOSSARIO TECNICO
═════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════

CommonJS: Formato JS do Node.js (require/module.exports, não ES6 import)
Express: Framework Web Node.js - gerencia rotas HTTP
REST API: Web services com GET/POST/PUT/DELETE
Middleware: Função que filtra ou processa requisições antesde chegar na rota
SQLite: Banco de dados leve, arquivo único, sem servidor
Schema: Estrutura das tabelas (colunas, tipos)
Migration: Script que altera schema (version control para BD)
Gemini API: API Google Generative AI para chamar modelo LLM
.env File: Arquivo com senhas/chaves (NÃO commitar para Git)
async/await: Syntax para operações assíncronas (Promises)
try/catch: Tratamento de erros em JavaScript
Port: Número que application escuta (default HTTP 80, HTTPS 443)
Localhost: IP 127.0.0.1 - computador local
require Cache: Node.js caches módulos após carregados (restart necessário para atualizar)
EOF: End of File
